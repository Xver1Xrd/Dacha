## О проекте

Сайт решает простую задачу: посетитель должен за несколько секунд понять, что делает бригада, увидеть реальных исполнителей с телефонами и отзывы клиентов, и позвонить. Админ должен без разработчика редактировать карту участков, список работников и отзывы через собственную панель, а не через правки кода.

Фронтенд — статические HTML/CSS/JS без сборки и фреймворков. Бэкенд — один Go-бинарник, который сам раздаёт статику и REST API поверх PostgreSQL (с файловым fallback для локальной разработки).

## Возможности

**Публичный сайт**
- Адаптивный лендинг со светлой/тёмной темой (сохраняется в `localStorage`)
- Карточки работников с телефоном, Telegram и счётчиком клиентов
- Бесконечная карусель отзывов (свайп/стрелки/точки/автопрокрутка) с бесшовным зацикливанием
- Плавающая кнопка звонка на мобильных, которая сама прячется там, где рядом уже есть кнопка звонка

**Админпанель** (`/admin.html`)
- Вход по логину/паролю, сессии в HttpOnly-cookie
- Редактор интерактивной карты участков (статус, кто работал, что делали)
- Управление работниками (имя, телефон, Telegram, счётчик клиентов, плашка «Новичок» первую неделю)
- Управление отзывами: добавление, редактирование, удаление, **изменение порядка** кнопками ▲/▼
- Управление другими администраторами с точечными правами доступа (карта / работники / администраторы / отзывы)

## Стек технологий

| Слой | Технологии |
|---|---|
| Бэкенд | Go 1.25, стандартный `net/http` (без веб-фреймворка), [`pgx/v5`](https://github.com/jackc/pgx) |
| Хранилище | PostgreSQL — или `data.json` на диске, если `DATABASE_URL` не задан |
| Фронтенд | Vanilla HTML/CSS/JS, без сборки, без npm-зависимостей |
| Тесты | Стандартный `testing`, `httptest` |
| CI/CD | GitHub Actions, деплой на Railway (Nixpacks) |

## Быстрый старт

Требуется Go 1.25+.

```bash
git clone https://github.com/Xver1Xrd/Dacha.git
cd Dacha
go run .
```

Сервер поднимется на `http://localhost:8080`. Открывайте сайт по этому адресу, не через `file://` — фронтенду нужен бэкенд рядом.

При первом запуске без `DATABASE_URL` рядом создастся `data.json` со стартовыми данными и учёткой администратора (логин и пароль — в `seed.go`, функция `seed()`).

Собрать бинарник:

```bash
go build -o dacha .
./dacha
```

Прогнать тесты:

```bash
go test ./...
```

## Переменные окружения

| Переменная | Обязательна | Назначение |
|---|---|---|
| `PORT` | нет | Порт сервера (по умолчанию `8080`) |
| `DATABASE_URL` | нет | Строка подключения к PostgreSQL. Если не задана — используется файловое хранилище `data.json` |

`Secure`-флаг у cookie и остальные HTTPS-зависимые вещи сервер определяет сам по `X-Forwarded-Proto` — ничего дополнительно настраивать не нужно, если вы за обратным прокси (Railway, Render, nginx и т.п.).

## API

Все мутирующие запросы (POST/PUT/DELETE) от авторизованного администратора обязаны дублировать значение cookie `snt_csrf` в заголовке `X-CSRF-Token`.

| Метод | Путь | Доступ | Назначение |
|---|---|---|---|
| GET | `/api/data` | все | аллеи, статусы участков, работники |
| GET | `/api/reviews` | все | список отзывов |
| POST | `/api/login` | все | вход, ставит cookie сессии и CSRF |
| POST | `/api/logout` | все | выход |
| GET | `/api/session` | все | проверка текущей сессии |
| GET / POST | `/api/admins` | admins | список / добавление администратора |
| DELETE | `/api/admins/{login}` | admins | удаление администратора |
| POST | `/api/workers` | workers | добавление работника |
| PUT / DELETE | `/api/workers/{id}` | workers | редактирование / удаление работника |
| POST | `/api/works` | map | статус и описание работ по участку (`rec:null` — сброс) |
| POST | `/api/reviews` | reviews | добавление отзыва |
| PUT / DELETE | `/api/reviews/{id}` | reviews | редактирование / удаление отзыва |
| POST | `/api/reviews/{id}/move` | reviews | сдвинуть отзыв на позицию вверх/вниз (`{"direction":"up\|down"}`) |

## Безопасность

- Пароли — PBKDF2-HMAC-SHA256 (соль + 100 000 итераций)
- Сессии с TTL 24 часа; в режиме Postgres хранятся в таблице `sessions` и переживают перезапуск процесса
- CSRF — double-submit cookie (`snt_csrf` + заголовок `X-CSRF-Token`)
- Rate limiting — 5 неудачных попыток входа за 15 минут на пару IP+логин, дальше блокировка на 15 минут
- Заголовки безопасности: CSP, `X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`, `Permissions-Policy`
- Лимиты длины на весь пользовательский ввод, тело запроса ограничено 1 МиБ

## Тесты и CI

```bash
go build ./...
go vet ./...
gofmt -l .
go test ./... -race
```

GitHub Actions гоняет этот же набор на каждый push и pull request. Тесты покрывают хеширование паролей, лимитер входа, сессии, файловое хранилище и полный HTTP-цикл «логин → CSRF → мутирующий запрос».

## Деплой

Проект деплоится на [Railway](https://railway.app) из ветки `master` — сборка через Nixpacks по `railway.json` (`go build -o dacha .`, healthcheck на `/api/data`). Если подключить плагин PostgreSQL, `DATABASE_URL` подхватится автоматически и сервер сам переключится в режим Postgres.

GitHub Pages не подходит — там только статика, а тут нужен постоянно работающий бэкенд.

## Что не доделано

- Публичная интерактивная карта участков — сейчас карта редактируется только из админки, на самом сайте её пока нет.
