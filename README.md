# СНТ «Михайловское» — сайт + бэкенд

Лендинг услуг по работе на участках с интерактивной картой работ и админпанелью.
Фронтенд — статика (HTML/CSS/JS), бэкенд — на Go (стандартная библиотека, без внешних зависимостей).

## Запуск

```bash
go run .
# или собрать бинарник:
go build -o dacha . && ./dacha
```

Сервер поднимется на http://localhost:8080 (порт можно сменить: `PORT=3000 go run .`).
Он раздаёт сайт и REST API. Открывайте сайт по адресу сервера, не через `file://`.

При первом запуске рядом создаётся `data.json` со стартовыми данными.
Стартовый логин/пароль администратора задаётся в `seed()` (`main.go` и `store.go`).

## Что где

- `index.html` — сайт (публичной карты пока нет — она не доработана и доступна только в админке).
- `admin.html` — админпанель (вход, редактор карты, работники, администраторы, отзывы).
- `css/`, `js/` — стили и фронтенд-логика. `js/store.js` — клиент API.
- `main.go` — сервер: статика + API. `data.json` — хранилище (создаётся автоматически, в git не коммитится).

## API (кратко)

| Метод | Путь | Доступ | Назначение |
|------|------|--------|------------|
| GET | `/api/data` | все | данные карты (аллеи, работы, работники) |
| POST | `/api/login` | все | вход (ставит cookie-сессию) |
| POST | `/api/logout` | все | выход |
| GET | `/api/session` | все | проверка текущей сессии |
| GET/POST | `/api/admins` · DELETE `/api/admins/{login}` | админ | управление администраторами |
| POST `/api/workers` · DELETE `/api/workers/{id}` | админ | управление работниками |
| POST | `/api/works` | админ | статус/инфо участка (`rec:null` — сброс) |

## Безопасность

- Пароли хранятся как PBKDF2-HMAC-SHA256 (соль + 100 000 итераций), не в открытом виде.
- Сессии — случайный токен в HttpOnly-cookie (хранятся в памяти, сбрасываются при перезапуске).
- **Для продакшена**: запускайте за HTTPS-прокси (nginx/Caddy) и включите флаг `Secure` у cookie
  (в `main.go`, в `http.SetCookie` при логине добавьте `Secure: true`).

## Хостинг

GitHub Pages не подходит (там только статика). Нужен хост, где можно запустить бинарник:
Render / Railway / Fly.io / собственный VPS. Соберите `go build` и запустите бинарник,
положив рядом записываемый `data.json`.
